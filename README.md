# critical-report-review

一个 [Claude Skill](https://docs.claude.com/en/docs/agents-and-tools/agent-skills/overview)，用批判性思维框架审查**咨询预测报告**、**公司财报**和**投行/卖方研究报告**，拆解逻辑谬误、隐含假设、证据薄弱环节和利益冲突——而不是简单做摘要。

## 这个 Skill 解决什么问题

把一份报告丢给 AI，默认得到的往往是一份"废话摘要"：复述报告说了什么，语气跟报告本身一样乐观。这个 Skill 让 Claude 代入不同的专业角色（企业首席战略官 / 首席风险官 / 逻辑学教授），对报告执行**"解构—审查—对抗"**的流程，而不是"阅读—复述"。

覆盖三类场景，各自使用一套针对性的审查框架：

| 报告类型 | 框架代号 | 核心角色 |
|---|---|---|
| 咨询机构行业趋势/预测报告 | 未来学家剃刀（The Futurist's Razor） | 企业首席战略官（CSO） |
| 公司财报（年报/季报/招股书） | 财报交叉审问（Cross-Examination） | 资深财务分析师 + 首席风险官（CRO） |
| 投行/券商卖方研究报告 | 阿尔法过滤器（Alpha Filter） | 首席风险官（CRO）+ 逻辑学教授 |

每套框架都会输出：区分事实与愿景/预测、挖掘隐含假设、评估证据来源与利益冲突、构建反例/替代解释（含"事前验尸"式压力测试）、给出一个明确的可信度判断。

## 安装方式

### Claude.ai / Claude Cowork（网页版技能）
1. 下载本仓库（或直接下载 Release 中的 `.skill` 文件）
2. 在 Claude.ai 的技能管理界面上传 `critical-report-review.skill`，或直接上传本仓库整个文件夹

### Claude Code
将 `critical-report-review/` 文件夹整体复制到你的技能目录（例如项目内的 `.claude/skills/` 或全局技能目录），Claude Code 会自动识别 `SKILL.md` 中的 frontmatter 并在合适场景下调用。

## 使用示例

上传一份咨询机构的行业预测报告、上市公司财报、或投行研报 PDF，然后说：

> "帮我深度分析一下这份报告，这个预测靠谱吗？"
> "这份研报的逻辑站得住脚吗？"
> "审查一下这份财报的风险点"

Claude 会自动识别报告类型并调用对应框架，输出结构化的批判性分析，而不是摘要。

## 设计理念

三套框架共享同一个通用审查骨架（详见 `SKILL.md`）：

1. **逻辑解构 / 语义排毒**：剥离大词和黑话，区分**事实**与**愿景**
2. **隐含假设审查**：报告成立需要相信哪些"未明说"的前提？是否激进？
3. **证据可信度**：证据来自客观数据还是主观信源（管理层指引/高管问卷）？是否存在利益冲突？
4. **反例与替代解释**：如果结论是错的，最可能的"死因"是什么？有没有更简单的解释？
5. **最终判决**：给出明确的可信度评分或分级判断
